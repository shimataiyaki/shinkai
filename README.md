# Shinkai

「組織の可観測性」を高めるための、実験的プロダクトシリーズ。

---

## 1. Shinkai とは

Shinkai は、**チームの情報の流れを見えるようにし、チーフの「確認疲れ」をなくす**ためのツール群です。

-   LINE で話したことが、自動でタスクとして記録される
-   チーフが「状況」と聞けば、今の進捗と止まっていることが返ってくる
-   Web でプロジェクトの計画と振り返りができる

---

## 2. 解決する問題

| 問題 | Shinkai の解決策 |
| :--- | :--- |
| 口頭やチャットで言ったことが消える | 会話からタスクを自動抽出し、記録する |
| 誰が何を抱えているかわからない | タスクの一覧と状態を可視化する |
| チーフが「どうなってる？」と何度も聞く | 「状況」と打てば、Bot が進捗と停滞を返す |
| プロジェクトがやりっぱなしで終わる | Web で振り返り、教訓を次のプロジェクトに活かす |

---

## 3. 全体アーキテクチャ

Shinkai は **Web（設計室）** と **アプリ（実行部隊）** の2層で構成されます。

```
┌─────────────────────────────────────────────┐
│                  Shinkai                     │
└─────────────────────────────────────────────┘
        │
        ├── Web（計画・設計・評価）
        │   ├── Vision Planner（ビジョン定義）
        │   ├── Conductor（権限設計）
        │   ├── Dashboard（状態可視化）
        │   └── Reflector（評価・振り返り）
        │
        └── アプリ（日常・実行・応答）
            ├── Connector for LINE（タスク抽出・状態応答）
            └── Connector for Voice（口頭記録・将来）
```

### 見えない背骨：統合レイヤー

ユーザーからは見えませんが、Web とアプリを `Shinkai ID` で接続し、データを一元管理する基盤です。

-   **Shinkai ID** (`proj-xxxx`) で全データを紐づけ
-   **共通データストア**（スプレッドシート）でタスク・権限・評価を一元管理
-   **状態応答エンジン** がチーフの問い合わせにリアルタイムで応答

---

## 4. プロダクト群

### Web（設計室）

| プロダクト | 役割 |
| :--- | :--- |
| **Vision Planner** | プロジェクトの「なぜ」と及第点を定義。Shinkai ID を発行。 |
| **Conductor** | タスクに権限を付与。「誰が何をどこまで決めていいか」を可視化。 |
| **Dashboard** | 進捗、タスク一覧、チームの状態を表示。Shinkai ID でログイン。 |
| **Reflector** | プロジェクト終了時に振り返り。自由記述から AI が教訓を抽出。 |

### アプリ（実行部隊）

| プロダクト | 役割 |
| :--- | :--- |
| **Connector for LINE** | LINE 上の会話からタスクを自動抽出・記録。「状況」と聞けば状態を返信。 |
| **Connector for Voice** (構想) | 口頭ミーティングを記録・文字起こしし、タスクを抽出。 |

---

## 5. データの流れ

```
[計画] → [実行] → [応答] → [評価] → [次の計画へ]
```

1.  **計画**：Vision Planner でプロジェクトを定義
2.  **実行**：Connector が日々の会話からタスクを抽出・記録
3.  **応答**：チーフの「状況」に Bot が進捗と停滞を返す
4.  **評価**：プロジェクト完了時に Reflector で振り返り
5.  **継承**：次のプロジェクトで過去の教訓を参照

---

## 6. 技術スタック

無料枠内での動作を前提とします。

| 要素 | 技術 |
| :--- | :--- |
| **Web ホスティング** | GitHub Pages |
| **バックエンド** | Google Apps Script (GAS) |
| **データストア** | Google スプレッドシート |
| **AI 抽出・分析** | Gemini API (無料枠) |
| **LINE 接続** | LINE Messaging API (Replyのみ、無料枠) |
| **認証** | Shinkai ID (`proj-xxxx`) |

## 7. 開発ロードマップ

1.  **Phase 1 (MVP)**：統合レイヤー基盤 & Connector for LINE の「状態応答」機能
2.  **Phase 2**：Vision Planner (Web) のプロトタイプ & Shinkai ID 連携
3.  **Phase 3**：Dashboard (Web) のプロトタイプ & Reflector による評価機能
4.  **Phase 4**：Conductor (権限設計) の実装
5.  **Phase 5**：Connector for Voice の検証

---

## 8. Shinkai の及第点

**チーフが Web でプロジェクトを立ち上げ、LINE に Connector を招待する。チームは普段通り会話し、タスクは自動で記録される。「状況」と聞けば進捗が返ってくる。プロジェクトが終われば Web で振り返り、教訓が次のプロジェクトに継承される。**
